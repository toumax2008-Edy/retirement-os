# Website Improvement Analyzer

企業ホームページ、LP、ECサイトを手動で診断し、改善レポートと営業提案書のたたき台を作るブラウザツールです。
依存ライブラリなしで、`index.html` を開くだけで使えます。

## Version

v1.0: 手動入力版

人がサイトを見て気づいた点を入力し、Markdown形式の改善レポートを生成します。
URL自動取得、SEO自動診断、AI API連携はまだ行いません。

## Purpose

受託営業、ポートフォリオ、自社事業の改善提案に使える「サイト診断の型」を作ります。
サイトを見ながら、良い点、改善点、CTA、スマホ対応、SEO、AIで改善できそうな点を整理できます。

## How To Use

1. `index.html` をブラウザで開く。
2. サイト名、URL、業種、サイト種別、ターゲットを入力する。
3. 第一印象、良い点、改善点、各診断メモを入力する。
4. 右側の Markdown レポートを確認する。
5. `Markdown をコピー` を押して、提案書、応募文、Notion、GitHub Issue などに貼る。

## Inputs

- サイト名
- URL
- 業種
- サイト種別
- ターゲット
- 第一印象
- 良い点
- 改善点
- デザイン
- 情報の分かりやすさ
- 問い合わせ導線
- スマホ対応
- 表示速度
- SEOの気づき
- AIで改善できそうな点
- 優先順位
- メモ

## Output

- サイト概要
- 良い点
- 改善点
- 優先順位
- AI活用の提案
- 次のアクション
- 提案書のたたき台

## Design Notes

- HTML単体で動く。
- 外部API、APIキー、ログインは使わない。
- 入力値はブラウザの localStorage に保存する。
- 入力項目は JavaScript の `fields` 配列で管理する。
- 将来AI連携しやすいよう、入力取得とMarkdown生成を分ける。

## Roadmap

- v1.5: SEO、スマホ対応、CTA、デザイン、表示速度のチェック項目強化。
- v2.0: 診断ロジック分離とAI API連携準備。ただしキーは安全に扱う。
- v3.0: URL入力から診断、改善案、提案書作成までの自動化。
