# Website Improvement Analyzer

企業ホームページ、LP、ECサイトを手動で診断し、改善レポートと営業提案書のたたき台を作るブラウザツールです。
依存ライブラリなしで、`index.html` を開くだけで使えます。

## Version

v1.5: 診断チェックリスト追加版

人がサイトを見て気づいた点を入力し、SEO、CTA、スマホ対応、ファーストビュー、信頼材料、表示速度のチェック結果を含むMarkdown形式の改善レポートを生成します。
URL自動取得、SEO自動診断、AI API連携はまだ行いません。

## Purpose

受託営業、ポートフォリオ、自社事業の改善提案に使える「サイト診断の型」を作ります。
毎回同じ観点でチェックできるようにして、提案品質のばらつきと診断漏れを減らします。

## How To Use

1. `index.html` をブラウザで開く。
2. サイト名、URL、業種、サイト種別、ターゲットを入力する。
3. 良い点、改善点、短期改善、中長期改善を入力する。
4. SEO、CTA、スマホ対応、ファーストビュー、信頼材料、表示速度をチェックする。
5. 右側の Markdown レポートを確認する。
6. `Markdown をコピー` を押して、提案書、応募文、Notion、GitHub Issue などに貼る。

## Inputs

- サイト概要
- 強み、良い点、改善点
- 優先順位と理由
- 今すぐ改善できること
- 中長期で改善すること
- SEOチェック
- CTAチェック
- スマホ対応チェック
- ファーストビューチェック
- 信頼材料チェック
- 表示速度チェックとメモ
- 最優先、次に直す点、余裕があれば直す点
- AIで改善できそうな点
- 提案トーン
- メモ

## Output

- サイト概要
- 強み、良い点、改善点
- 優先順位と理由
- 診断チェックリスト
- 改善優先度
- 今すぐ改善できること
- 中長期で改善すること
- 診断メモ
- AI活用の提案
- 提案書の短い版
- 提案書の詳しい版

## v1.5 Improvements

- SEO、CTA、スマホ対応、ファーストビュー、信頼材料、表示速度のチェックリストを追加。
- チェック結果をMarkdownレポートへ反映。
- 改善優先度を「最優先」「次に直す」「余裕があれば直す」に分けて出力。
- 営業でそのまま使いやすい短い提案文を追加。
- 既存の詳しい提案文にチェックリスト結果を反映。

## Design Notes

- HTML単体で動く。
- 外部API、APIキー、ログインは使わない。
- 入力値はブラウザの localStorage に保存する。
- 入力項目は JavaScript の `fields` 配列で管理する。
- チェック項目は `checklistGroups` 配列で管理する。
- 将来AI連携しやすいよう、入力取得とMarkdown生成を分ける。

## Roadmap

- v1.6: チェック結果から簡易スコアを出す。
- v2.0: 診断ロジック分離とAI API連携準備。ただしキーは安全に扱う。
- v3.0: URL入力から診断、改善案、提案書作成までの自動化。
