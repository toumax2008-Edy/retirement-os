# Roadmap

引退OSは、案件選別・受託実績・自社事業化の実験環境として育てます。
最初から大きく作らず、小さく使える道具を増やしていきます。

## Phase 1: Public Ready

- GitHub に置ける基本ファイルをそろえる。
- README、LICENSE、運用ルール、ロードマップを整える。
- Opportunity Scorecard を実務で使える状態にする。

## Phase 2: Work Logs

- 案件レビューのテンプレートを使って判断履歴を残す。
- Scorecard のサンプルを増やす。
- 受託、ポートフォリオ、自社事業の判断基準を比較できるようにする。

## Phase 3: Tool Expansion

- Client CRM で応募・受注案件と次アクションを管理する。
- Work Log Builder で日々の作業を記録し、実績化候補を抽出する。

- Proposal Draft Builder を使い、応募文の下書きを実案件で検証する。
- Portfolio Case Builder を使い、完了案件を実績紹介文へ変換する。
- 自社事業アイデアの検証チェックリストを追加する。

## Phase 4: Automation

- Markdown の記録を一覧化する。
- GitHub Issue や Google Sheets へ移せる形を検討する。
- 外部 API を使う場合は、認証情報を保存しない設計にする。

## Current Priority

1. Opportunity Scorecard を継続利用する。
2. 実際の案件レビューを 3 件以上記録する。
3. 判断しやすい評価項目へ少しずつ直す。
