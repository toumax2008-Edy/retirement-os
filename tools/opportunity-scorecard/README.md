# Opportunity Scorecard

案件、ポートフォリオ企画、自社事業アイデアを同じ基準で比較するための小型ブラウザツールです。
依存関係なしで `index.html` を開くだけで使えます。

## Purpose

思いつきや緊急度だけで動かず、売上性、資産化、実行容易性、リスクを一枚で見て優先順位を決めます。
受託の案件選定、ポートフォリオに載せる制作物の選定、自社事業の初期仮説整理へ転用できます。

## How To Use

1. `index.html` をブラウザで開く。
2. 案件名、評価日、用途、概要を入力する。
3. 各評価項目を 1 から 5 で採点する。
4. 理由メモと判断メモを書く。
5. 応募 / 保留 / 見送り の判定、合計点、次のアクションを確認する。
6. Markdown をコピーして、提案書、日報、GitHub Issue、Notion などに貼る。

## Inputs

- Name: 案件名、施策名、プロダクト名
- Review Date: 評価日
- Type: 受託、ポートフォリオ、自社事業、共通
- Summary: 何を改善するか
- Scores: 売上性、再利用性、実行速度、証明価値、戦略性、リスク低さ、作業負荷の低さ
- Reason: 応募、保留、見送りにした理由
- Notes: 判断材料、懸念、次の一手

## Outputs

- 合計スコア（35点満点）
- 応募 / 保留 / 見送り の判定
- Markdown 形式の記録

## Examples

- `examples/sample-job-review.md`: 受託案件レビューのサンプル。

## Decision Rule

- 28点以上: 応募
- 21点から27点: 保留
- 20点以下: 見送り

## Long-Term Design

- 最初は静的 HTML として保守コストを低くする。
- 評価軸を増やす場合は、先に README の用途を更新する。
- データ保存が必要になったら、JSON export/import を追加する。
- チーム運用が必要になったら、GitHub Issue テンプレートまたは Google Sheets 連携へ拡張する。

## Reuse Targets

- 受託: 提案すべき案件、断る案件、追加提案の判断。
- ポートフォリオ: 見せる価値の高い制作物の選定。
- 自社事業: MVP 候補、検証順序、撤退判断の整理。
