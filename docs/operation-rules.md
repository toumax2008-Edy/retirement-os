# Operation Rules

引退OSを長く使うための基本ルールです。

## Purpose

このリポジトリは、案件選別・受託実績・自社事業化の実験環境です。
迷った案件や企画を、あとから見返せる形で残します。

## Add Small

- まず小さく作る。
- 使ってから直す。
- 1つのツールに目的を詰め込みすぎない。

## Keep Records

- 判断した理由を短く残す。
- 点数だけで決めず、理由メモも残す。
- うまくいかなかった案も、学びがあれば `archive/` に残す。

## Safety

- 個人情報、認証情報、契約情報、支払い情報は置かない。
- `.env` は Git 管理しない。
- 公開、契約、支払い、認証が必要な作業は、実行前に確認する。

## GitHub

- `main` は見せられる状態にする。
- 1回の変更は小さくする。
- README が古くなったら、コードと一緒に直す。

## Review Timing

- 新しいツールを追加した時。
- 外部公開前。
- 使い方が変わった時。
- 1か月以上使っていないものを整理する時。
